# CMDB 模型唯一规则升级 - Tech Plan

日期：2026-03-26

## 技术目标与非目标

### 技术目标

- 在现有 CMDB 架构内落地“模型级额外唯一规则”能力，不新增服务，不改造 CMDB 之外模块。
- 保留 inst_name 内置唯一能力与历史字段级 is_only 后端兼容能力。
- 在模型实体上新增独立属性 unique_rules，采用 JSON 字符串持久化，默认值为 "[]"，不嵌入 attrs。
- 新增模型级唯一规则 CRUD 接口，并统一接入实例新增、修改、复制、导入、批量更新链路。
- 新增字段唯一性只读展示字段 unique_display_type，统一前后端展示口径。
- 新增模型复制、模型配置导入导出对 unique_rules 的支持。
- 所有规则保存操作必须扫描历史实例，发现冲突则阻止保存。
- 规则冲突提示统一为“规则序号 + 字段中文名组合 + 冲突值摘要”。

### 非目标

- 不替换、移除、弱化 inst_name 的内置唯一能力。
- 不恢复前端对 is_only 的新增或编辑能力。
- 不支持条件唯一、跨模型唯一、按组织范围唯一。
- 不支持超过 3 条额外唯一规则。
- 不支持 enum、tag、bool 字段参与额外唯一规则。
- 不改造权限系统、审批流、搜索系统、变更记录系统的通用架构。

## 2) 核心数据结构 / Schema 定义

### 2.1 Python Dataclasses

```python
from dataclasses import dataclass, field
from typing import Any, Literal

UniqueDisplayType = Literal["none", "single", "joint"]


@dataclass(slots=True, frozen=True)
class ModelUniqueRule:
    """
    模型级唯一规则持久化结构。
    存储位置：model.unique_rules（JSON string）
    约束：
    1. rule_id 使用 uuid.uuid4().hex 生成
    2. order 为 1-based 连续整数，由服务端重排
    3. field_ids 长度 >= 1
    4. field_ids 内元素不可重复
    """
    rule_id: str
    order: int
    field_ids: list[str]


@dataclass(slots=True, frozen=True)
class UniqueRulePayload:
    """
    前端提交的规则载荷。
    创建和编辑都只允许传 field_ids。
    rule_id 和 order 由服务端生成或维护。
    """
    field_ids: list[str]


@dataclass(slots=True, frozen=True)
class UniqueRuleFieldMeta:
    """
    可参与唯一规则的字段元数据。
    disabled_reason 仅用于前端候选展示。
    """
    attr_id: str
    attr_name: str
    attr_type: str
    is_required: bool
    selectable: bool
    disabled_reason: str = ""


@dataclass(slots=True, frozen=True)
class UniqueRuleConflict:
    """
    唯一规则冲突结果。
    用于规则保存校验、实例写入校验、导入校验。
    """
    rule_id: str
    rule_order: int
    field_ids: list[str]
    field_names: list[str]
    field_values: dict[str, Any]
    exist_instance_ids: list[int]
    exist_instance_names: list[str]
    message: str


@dataclass(slots=True)
class UniqueRuleCheckContext:
    """
    唯一规则检查上下文。
    - attrs_by_id: 当前模型字段定义映射
    - unique_rules: 额外唯一规则
    - builtin_unique_fields: 必含 inst_name
    - legacy_unique_fields: 历史 is_only 字段集合
    """
    model_id: str
    attrs_by_id: dict[str, dict[str, Any]]
    unique_rules: list[ModelUniqueRule] = field(default_factory=list)
    builtin_unique_fields: set[str] = field(default_factory=set)
    legacy_unique_fields: set[str] = field(default_factory=set)


@dataclass(slots=True, frozen=True)
class AttrUniqueDisplay:
    """
    字段唯一性只读展示结构。
    返回给属性列表接口。
    """
    attr_id: str
    is_only: bool
    unique_display_type: UniqueDisplayType
```

### 2.3 属性列表接口返回扩展

```python
# 现有字段结构保留 is_only
# 新增只读字段 unique_display_type

{
    "attr_id": "serial_no",
    "attr_name": "序列号",
    "attr_type": "str",
    "is_only": False,
    "is_required": True,
    "editable": True,
    "unique_display_type": "single"
}
```

unique_display_type 规则：

- none：既不是历史 is_only，也未被额外唯一规则使用
- single：历史 is_only 为 true，或被单字段额外唯一规则使用
- joint：被多字段额外唯一规则使用

### 2.4 导入导出 Sheet 结构

不新增独立规则 sheet，直接复用现有 attr-{model_id} sheet，并在属性定义列后追加唯一规则归属列：

```text
unique_rule_order
```

字段说明：

- unique_rule_order：可空正整数。
- 空值：该字段不参与额外唯一规则。
- 同一个正整数：表示这些字段属于同一条额外唯一规则。
- 单字段规则：仅 1 行字段带相同 unique_rule_order。
- 多字段规则：多行字段带相同 unique_rule_order。

导出示例：

```text
attr_id      | attr_name | ... | unique_rule_order
ip_addr      | IP地址    | ... | 1
cloud        | 云区域    | ... | 1
serial_no    | 序列号    | ... | 2
owner        | 负责人    | ... |
```

导入规则：

- 仅识别 attr-{model_id} sheet 中的 unique_rule_order 列。
- 服务端按 unique_rule_order 分组收集 field_ids。
- 分组数必须 <= 3。
- 每个分组至少包含 1 个字段。
- 导入后按 unique_rule_order 升序重排为连续 1-based order。
- rule_id 不从 Excel 导入，统一由服务端重新生成 uuid.uuid4().hex。

## 4) 核心逻辑伪代码 (Step-by-Step Logic)

### 4.1 规则保存

```text
1. 入口：
   - create_unique_rule / update_unique_rule
   - 输入只有 field_ids
   - 先 build_unique_rule_context(model_id)

2. 校验 payload 基础合法性：
   2.1 field_ids 必须是非空数组
   2.2 field_ids 内不能重复
   2.3 若 editing_rule_id 为空，则现有规则数必须 < 3
   2.4 每个字段必须存在于 attrs_by_id
   2.5 field_id 不允许等于 inst_name
   2.6 字段必须 is_required=True
   2.7 attr_type 不允许是 enum/tag/bool

3. 校验跨规则字段复用：
   3.1 收集其他规则中所有 field_ids
   3.2 若 payload 中任一 field_id 已被其他规则使用，则失败

4. 生成下一版规则列表：
   4.1 create:
       - rule_id = uuid.uuid4().hex
       - order = len(rules) + 1
       - append 新规则
   4.2 update:
       - 按 rule_id 替换原规则 field_ids
       - 保留原 order
   4.3 reorder_unique_rules()

5. 历史实例冲突扫描：
   5.1 查询当前模型全部实例
   5.2 调用 validate_unique_rules_against_existing_instances()
   5.3 若返回 conflicts 非空，则直接失败，不写入 model.unique_rules

6. 落库：
   6.1 model.unique_rules = dump_unique_rules(next_rules)
   6.2 GraphClient.set_entity_properties(MODEL, ...)
   6.3 返回最新规则列表 + candidate_fields
```

### 4.2 实例写入校验

```text
1. 进入实例写入入口：
   - instance_create
   - instance_update
   - batch_instance_update
   - inst_import_support_edit

2. 构建统一上下文：
   2.1 ctx = build_unique_rule_context(model_id)
   2.2 attrs = search_model_attr_v2(model_id)
   2.3 保留现有 _build_check_attr_map() 的 is_only / is_required / editable

3. 生成待校验候选行：
   3.1 create:
       - candidate_rows = [instance_info]
   3.2 update:
       - current = 当前实例
       - merged = merge(current, update_data)
       - candidate_rows = [merged]
       - exclude_instance_ids = {current_id}
   3.3 batch_update:
       - 对每个 inst_id 读取 current
       - merged = merge(current, update_data)
       - candidate_rows = all merged rows
       - exclude_instance_ids = set(all current ids)
   3.4 import:
       - 解析 Excel 得到 normalized rows
       - candidate_rows = normalized rows

4. 执行现有单字段唯一：
   4.1 inst_name 一定参与
   4.2 历史 is_only 字段照旧参与
   4.3 复用 check_unique_attr()

5. 执行新增模型级规则唯一：
   5.1 check_unique_rules(candidate_rows, ctx.unique_rules, exist_items, ...)
   5.2 先检查 candidate_rows 与 exist_items 的组合冲突
   5.3 再检查 candidate_rows 之间的批次内冲突
   5.4 任一冲突，抛业务异常

6. 冲突提示构造：
   6.1 rule label = f"规则 {order}【{' + '.join(field_names)}】"
   6.2 value summary = 按 field_names 输出 key=value
   6.3 message = f"{rule label} 与现有实例冲突：{value summary}"

7. 校验通过后继续原写入逻辑
```

### 4.3 字段变更阻断

```text
1. 入口：
   - delete_model_attr
   - update_model_attr 中涉及 is_required/attr_type 变化的分支

2. 查询 model.unique_rules
3. 若 target_attr_id 不在任何规则中：
   - 直接放行

4. 若 operation == delete：
   - 直接阻止
   - message = "字段已被唯一规则使用，请先删除相关唯一规则"

5. 若 operation == update_required：
   - 若 next_attr.is_required=False，则阻止
   - message = "字段已被唯一规则使用，不能取消必填"

6. 若 operation == update_type：
   - 若 next_attr.attr_type in {enum, tag, bool}，则阻止
   - 否则放行

7. update_model_attr 内如果 attr_id 本身未变，但 attr_type / is_required 变更触发冲突，也必须中断整个保存
```

### 4.4 模型复制与导入导出

```text
A. 模型复制
1. copy_model(copy_attributes=False):
   - 新模型 unique_rules 固定写 "[]"
2. copy_model(copy_attributes=True):
   - 复制 attrs
   - 复制 unique_rules
   - 在目标 attrs 上重新执行 validate_unique_rule_payload()
   - 若字段集合法则不满足则中断复制
3. 因新模型无实例，不做历史实例冲突扫描

B. 模型配置导出
1. 对每个模型读取 model.unique_rules
2. 继续复用 attr-{model_id} sheet，不新增 unique_rules sheet
3. 在现有属性列尾部追加 unique_rule_order 列
4. 对每条规则中的字段，在对应属性行写入相同的 unique_rule_order
5. 不导出 rule_id；导入时由服务端重新生成

C. 模型配置导入
1. 遍历 attr-{model_id} sheet 时同步读取 unique_rule_order 列
2. 过滤 unique_rule_order 为空的字段行
3. 按 unique_rule_order 分组生成 field_ids
4. unique_rule_order 必须是正整数，分组数必须 <= 3
5. 在目标模型 attrs 上执行规则合法性校验
6. 若目标模型已有实例，再执行历史实例冲突扫描
7. 校验通过后写入 model.unique_rules
```
