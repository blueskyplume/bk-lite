# CMDB数据订阅

日期：2026-03-10

## 背景

当前 CMDB 对实例数据变化缺少主动通知能力，负责人通常依赖人工巡检或被动发现，存在以下问题：

- 关键配置被变更后无法及时感知（例如物理服务器内存配置变化）。
- 关联资源变化难以及时追踪（例如关联磁盘容量变化）。
- 时间类字段接近到期时缺少提前提醒（例如维保时间临近）。

为提升 PLG 场景下的自助体验与可用性，CMDB 需要提供“数据订阅”能力：用户可按模型与规则订阅实例数据变化，并通过系统管理已有通知渠道接收提醒。

## 需求描述

CMDB 新增“数据订阅”能力，支持在当前组织范围内订阅实例数据变化并发送通知。

### 1）订阅入口与范围

- 在资产页面新增“数据订阅”按钮，点击后以侧边弹框展示当前用户可见的订阅规则列表。
- 订阅规则的创建、查看、编辑、启停、删除均在该侧边弹框内完成。
- 订阅规则作用范围限定为当前组织，不包含子组织。
- 规则新增“所属组织”字段用于权限控制：仅当规则所属组织与当前用户组织匹配时，当前用户才可管理该规则。

Nano Banana 提示词：

- 设计一个 CMDB 资产页面，页面右上角有“数据订阅”主按钮。点击后右侧滑出抽屉，标题为“数据订阅规则”。抽屉内展示规则列表（规则名、所属组织、目标模型、状态、最近触发时间、操作列）。操作列包含查看、编辑、启停、删除。列表顶部有“新建规则”按钮和搜索框。页面风格企业级、信息密度中等、操作清晰。
- 在同一个原型中补充权限态：当用户组织与规则所属组织不一致时，编辑、启停、删除按钮置灰并显示 tooltip“仅所属组织可管理”。

### 2）快速订阅入口

提供以下四个快速订阅入口，减少用户配置成本：

- 实例列表多选后提供“订阅”按钮，点击后默认选择已勾选实例；规则名称默认 `{模型}{当前时间戳}`；通知对象默认当前用户；所属组织默认当前组织。
- 实例列表筛选后提供“订阅”按钮，点击后默认按当前筛选条件选择实例；规则名称默认 `{模型}{当前时间戳}`；通知对象默认当前用户；所属组织默认当前组织。
- 实例详情页提供“订阅”按钮，点击后默认选择当前实例；规则名称默认 `{实例名称}{当前时间戳}`；通知对象默认当前用户；所属组织默认当前组织。
- 资产页面“数据订阅”侧边弹框提供完整规则管理入口（增删改查）。

Nano Banana 提示词：

- 设计四个“快速订阅”入口联动原型：1）实例列表多选后显示“订阅”按钮；2）实例列表筛选后显示“订阅”按钮；3）实例详情页显示“订阅”按钮；4）从“数据订阅规则”抽屉点击“新建规则”。所有入口打开同一个“新建订阅规则”抽屉，但默认值不同：多选入口默认已选实例；筛选入口默认按当前筛选条件选实例；详情入口默认当前实例；规则名称默认“模型或实例名+时间戳”；通知对象默认当前用户；所属组织默认当前组织。请把默认值以高亮标签形式展示，便于用户理解。

### 3）支持的触发类型

订阅规则支持以下触发类型：

- 属性变化：目标模型实例指定字段值发生变化时触发；当实例新增进入订阅范围或已在订阅范围内的实例被删除时，也按属性变化类通知发送“实例新增 / 实例删除”摘要。
- 关联变化：目标模型实例一跳关联的另一模型实例数据发生变化时触发。
- 临近到期：目标模型时间类型字段值在“到期前 N 天”范围内时触发（按自然日口径）；该类型通知按自然日去重并按日聚合，不适用 1 分钟聚合窗口。

其中，属性变化、关联变化及“实例新增 / 实例删除”摘要属于其他数据变化，可在同一规则的 1 分钟窗口内合并为一次通知；临近到期需单独通知。

Nano Banana 提示词：

- 设计“触发类型”配置区，使用单选卡片组件展示三种类型：属性变化、关联变化（一跳）、临近到期。切换卡片时右侧动态出现对应配置表单：属性变化显示字段多选；关联变化显示关联模型下拉+关联字段多选；临近到期显示时间字段下拉+提前天数输入框并标注“自然日”。在同一画布给出三种切换后的状态对比。

### 4）规则配置要求

创建/编辑订阅规则时需支持以下配置：

- 规则名称（必填）。
- 所属组织（必填，默认当前组织，用于规则管理权限控制）。
- 目标模型（必选）。
- 选择实例（必选）：
  - 支持逐个勾选实例。
  - 支持按搜索条件筛选实例。
  - 搜索条件最多支持 8 个，多条件之间为“且（AND）”逻辑。
- 触发类型（必选，支持上述三类）。
- 触发条件：
  - 属性变化：可选择监听字段。
  - 关联变化：可选择一跳关联模型与监听字段。
  - 临近到期：可设置提前天数（正整数）。
- 接收对象（至少一个，用户或用户组）。
- 通知渠道（必选，使用系统管理已有通知渠道）。
- 规则状态（启用/停用）。


### 5）通知发送与防打扰

- 触发后按规则向接收对象发送通知。
- 内置防打扰聚合策略区分为两类：属性变化、关联变化及“实例新增 / 实例删除”摘要作为其他数据变化，在同一规则的 1 分钟窗口内可聚合为一次通知，避免通知风暴。
- 临近到期在同一规则下按自然日去重并按日聚合发送；同一实例的同一到期字段在同一自然日内最多发送 1 条提醒。
- 同一规则在相近时间内同时出现临近到期与其他数据变化时，需分开通知，不将两类内容合并为同一条通知。
- 通知的统一消息结构、触发类型差异与数量级处理规则按第 6 节执行。


### 6）通知消息设计

- 通知采用统一消息结构：标题、正文、触发时间、聚合说明。正文至少包含模型名称、实例标识、触发类型、变化摘要（或到期信息）。
- 标题需优先表达触发类型与影响范围：
  - 仅 1 个实例触发时，标题包含实例标识。
  - 聚合多个实例时，标题包含模型名称与受影响实例数量。
  - 当同一 1 分钟窗口内同时包含属性变化、关联变化或实例范围变化时，标题可使用“数据变化”作为统一表达。
  - 其他数据变化合并通知时，正文需保留每条摘要对应的具体变化类型或变化对象。
- 属性变化通知需展示实例标识、变化字段名称、变化前值、变化后值；变化摘要优先使用符号表达，如“字段名: 旧值 -> 新值”；同一实例在 1 分钟窗口内发生多个字段变化时，合并为同一实例摘要。
- 实例新增提示需展示实例标识、进入订阅范围的原因或匹配条件摘要；实例删除提示需展示实例标识、删除前的关键标识信息与删除时间；两类提示不使用“变化前值 / 变化后值”表达。
- 关联变化通知需展示主实例标识、关联模型名称、关联实例标识或关联对象摘要，并说明关联对象是新增、删除还是属性更新。
- 临近到期通知需展示实例标识、到期字段名称、到期日期、剩余天数；不使用“变化前值 / 变化后值”表达。
- 当规则按筛选条件选实例时，新实例首次满足筛选条件并进入订阅范围，需发送“实例新增”提示；当已在订阅范围内的实例被删除时，需发送“实例删除”提示。
- 不同实例变化数量级下的通知内容按以下规则处理：
  - 单实例触发：展示该实例的完整变化摘要或到期信息。
  - 2 至 5 个实例触发：逐条列出每个实例的简要摘要。
  - 超过 5 个实例触发：正文仅展示前 5 个实例摘要，并补充“另有 N 个实例发生同类变化”的汇总说明。
- 同一规则在不同聚合窗口内若同时出现多种类型变化，按以下规则通知：
  - 临近到期仅与临近到期聚合，且聚合粒度为自然日，不参与 1 分钟窗口聚合。
  - 属性变化、关联变化及“实例新增 / 实例删除”摘要可在同一 1 分钟窗口内合并为一次数据变化通知，并在正文中分项展示。
- 聚合通知需展示本次聚合的实例总数与触发时间范围；单条通知展示单一触发时间。临近到期按日聚合时，触发时间范围可展示为当日扫描时间或当日提醒日期。
- 变化摘要应以用户可读文本展示，不直接输出难以理解的原始结构；字段值过长时可摘要化呈现，但需保留关键变化信息。
- 变化摘要可采用以下符号约定简化表达：
  - 属性变化使用“旧值 -> 新值”。
  - 实例新增使用“+ 实例标识”或“+ 实例标识（匹配条件摘要）”。
  - 实例删除使用“- 实例标识”或“- 实例标识（删除前关键标识）”。
  - 关联对象新增、删除、更新可分别使用“+ 关联对象”“- 关联对象”“关联字段: 旧值 -> 新值”。
  - 临近到期可使用“字段名: 到期日期（剩余 N 天）”。

场景示例：

- 场景一：单实例属性变化。示例通知可表达为：标题“物理服务器 srv-prod-01 属性变化”；正文“模型：物理服务器；实例：srv-prod-01；触发类型：属性变化；变化摘要：内存容量: 64GB -> 128GB；触发时间：2026-03-18 10:15:00”。
- 场景二：同一实例多个字段在 1 分钟内连续变化。示例通知可表达为：标题“物理服务器 srv-prod-01 属性变化”；正文“模型：物理服务器；实例：srv-prod-01；触发类型：属性变化；变化摘要：CPU 核数: 16 -> 32，内存容量: 64GB -> 128GB；触发时间范围：2026-03-18 10:15:00 至 2026-03-18 10:15:42”。
- 场景三：3 个实例在 1 分钟内发生同类属性变化。示例通知可表达为：标题“物理服务器 3 个实例属性变化”；正文“模型：物理服务器；触发类型：属性变化；变化摘要：1）srv-prod-01：内存容量: 64GB -> 128GB；2）srv-prod-02：内存容量: 64GB -> 128GB；3）srv-prod-03：内存容量: 64GB -> 128GB；触发时间范围：2026-03-18 10:15:00 至 2026-03-18 10:15:58”。
- 场景四：8 个实例在 1 分钟内发生同类属性变化。示例通知可表达为：标题“物理服务器 8 个实例属性变化”；正文“模型：物理服务器；触发类型：属性变化；变化摘要：1）srv-prod-01：内存容量: 64GB -> 128GB；2）srv-prod-02：内存容量: 64GB -> 128GB；3）srv-prod-03：内存容量: 64GB -> 128GB；4）srv-prod-04：内存容量: 64GB -> 128GB；5）srv-prod-05：内存容量: 64GB -> 128GB；另有 3 个实例发生同类变化；触发时间范围：2026-03-18 10:15:00 至 2026-03-18 10:15:59”。
- 场景五：关联变化。示例通知可表达为：标题“物理服务器 srv-prod-01 关联对象变化”；正文“模型：物理服务器；实例：srv-prod-01；触发类型：关联变化；变化摘要：disk-data-01 容量: 500GB -> 1000GB；触发时间：2026-03-18 11:20:00”。
- 场景六：临近到期。示例通知可表达为：标题“物理服务器 srv-prod-01 临近到期提醒”；正文“模型：物理服务器；实例：srv-prod-01；触发类型：临近到期；到期信息：维保到期时间: 2026-03-25（剩余 7 天）；触发时间：2026-03-18 09:00:00”。同一实例的同一到期字段在 2026-03-18 当天不重复提醒，下一自然日如仍满足条件再发送新通知。
- 场景七：筛选订阅范围内出现新增实例。示例通知可表达为：标题“物理服务器出现新增实例”；正文“模型：物理服务器；实例：srv-prod-09；触发类型：属性变化；变化摘要：+ srv-prod-09（机房=广州，状态=运行中）；触发时间：2026-03-18 13:05:00”。
- 场景八：已订阅实例被删除。示例通知可表达为：标题“物理服务器 srv-prod-03 已删除”；正文“模型：物理服务器；实例：srv-prod-03；触发类型：属性变化；变化摘要：- srv-prod-03（主机名=srv-prod-03，IP=10.0.0.23）；触发时间：2026-03-18 13:20:00”。
- 场景九：同一规则在 1 分钟内同时出现属性变化和关联变化。示例通知可合并为一条：标题“物理服务器 3 个实例数据变化”；正文“模型：物理服务器；触发类型：属性变化、关联变化；变化摘要：1）[属性变化] srv-prod-01：内存容量: 64GB -> 128GB；2）[关联变化] srv-prod-02：关联对象 disk-data-01 容量: 500GB -> 1000GB；3）[属性变化] srv-prod-03：CPU 核数: 16 -> 32；触发时间范围：2026-03-18 10:15:00 至 2026-03-18 10:15:58”。
- 场景十：同一规则在 1 分钟内同时出现实例新增和普通字段变化。示例通知可合并为一条：标题“物理服务器 4 个实例数据变化”；正文“模型：物理服务器；触发类型：属性变化；变化摘要：1）[实例新增] + srv-prod-09（机房=广州，状态=运行中）；2）[属性变化] srv-prod-01：内存容量: 64GB -> 128GB；3）[属性变化] srv-prod-02：CPU 核数: 16 -> 32；4）[属性变化] srv-prod-03：主机名称: old-name -> new-name；触发时间范围：2026-03-18 13:05:00 至 2026-03-18 13:05:42”。
- 场景十一：临近到期与其他数据变化同时触发。示例通知应拆分为两条：第一条为临近到期通知，标题“物理服务器 srv-prod-01 临近到期提醒”，正文展示到期信息；第二条为数据变化通知，标题“物理服务器 3 个实例数据变化”，正文展示变化摘要。


### 7）实例数据校验与变更口径

- 属性变化与关联变化按 CMDB 当前已生效数据变化事实触发。
- 临近到期提醒按规则当前绑定字段与当前实例值计算。
- 公共选项库、枚举、多选等字段在本需求中按现有 CMDB 字段行为处理，不单独改变其业务语义。


### 8）规则生效与历史数据

- 订阅规则保存后仅对后续触发生效，不追补历史变更通知。
- 临近到期规则在启用后按调度周期执行扫描并触发。
- 历史实例值本身不因订阅能力而被修改。

## 适用范围

- CMDB：资产页面“数据订阅”侧边弹框与快速订阅入口。
- CMDB：数据订阅规则管理（创建、查看、编辑、启停、删除）。
- CMDB：实例变化触发与通知发送。
- 系统管理：复用已有通知渠道进行消息投递。

## 非目标（本期不包含）

- 跨组织/跨租户订阅。
- 多跳关联链路订阅（仅支持一跳关联）。
- 新增通知渠道类型（仅复用系统管理已有渠道）。
- 复杂流程能力（如审批流、版本管理、订阅模板市场）。

## 验收标准

- 资产页面存在“数据订阅”按钮，点击后可在侧边弹框查看并管理当前用户可见规则。
- 规则作用范围仅为当前组织，其他组织不可见不可用。
- 规则包含“所属组织”字段，且仅所属组织匹配当前用户时可管理。
- 提供四个快速订阅入口：实例列表多选、实例列表筛选、实例详情、管理侧边弹框。
- 快速订阅入口默认值生效：实例范围、规则名称、通知对象、所属组织符合预期。
- 规则支持三类触发：属性变化、关联变化（一跳）、临近到期（按自然日）。
- 规则配置项完整可用：名称、所属组织、目标模型、选择实例、触发类型与条件、接收对象、通知渠道、启停状态。
- 选择实例支持逐个勾选与条件筛选；条件数量最多 8 个且按 AND 逻辑生效。
- 触发后可通过系统管理已有通知渠道向接收对象发送通知。
- 通知采用统一消息结构，单条通知与聚合通知均可识别模型、实例或实例数量、触发类型、摘要、时间。
- 属性变化、关联变化、临近到期三类通知的摘要格式明确，用户可区分通知来源与变化类型。
- 实例新增进入订阅范围、实例删除退出订阅范围时可发送可识别的摘要提示，且不要求使用前后值表达。
- 属性变化、关联变化及“实例新增 / 实例删除”摘要可在同一规则的 1 分钟窗口内聚合为一次通知，不产生明显重复轰炸。
- 临近到期通知按自然日去重并按日聚合；同一实例的同一到期字段在同一自然日内最多提醒一次。
- 临近到期与其他数据变化分开通知，不混合展示不同类别摘要。
- 同一规则在同一 1 分钟窗口内出现属性变化、关联变化或“实例新增 / 实例删除”摘要时，可汇总为一次其他数据变化通知，并在正文中区分具体变化类型。
- 同一规则在相近时间内同时出现临近到期与其他数据变化时，会拆分为两类通知分别发送。
- 聚合通知按实例数量分层展示：单实例展示完整明细，2 至 5 个实例逐条摘要，超过 5 个实例展示前 5 条并说明其余数量。
- 停用规则后不再触发通知，重新启用后恢复触发。
- 规则仅对创建后触发生效，不追补历史变更通知。
