# 监控系统 集成-新增API模版与自定义API模版需求

- 文档日期：2026-03-26
- 所属模块：监控系统 / 集成 / 模版
- 需求类型：新增功能
- 参考原型：
  - spec/prototype/监控系统/集成/集成.html
  - spec/prototype/监控系统/集成/模版/自定义API模版.html

## 1. 背景与问题

- 当前监控系统集成能力以内置模版为主，适合标准化场景，但无法覆盖用户通过业务接口主动上报监控数据的自定义接入诉求。
- 现有监控接入缺少一类“通过 API 推送指标”的模版形态，导致非 Agent、非周期采集类数据难以纳入统一监控体系。
- 对于 API 接入场景，用户不仅需要一个模版入口，还需要完整的接入说明、组织与秘钥绑定、资产选择、示例代码和指标定义能力，才能完成实际接入。
- 原型已体现“集成列表新增自定义 API 模版入口”和“自定义 API 模版配置页”两段链路，需要沉淀为可研发、可测试、可验收的需求文档。

## 2. 目标

1. 在监控系统“集成”模块中新增自定义 API 模版能力，作为一种新的接入模版类型提供给用户。
2. 支持用户基于组织和 API 秘钥完成 API 数据接入配置，并绑定需要接入的监控资产。
3. 支持为自定义 API 模版维护指标分组和指标定义，形成从数据接入到指标管理的完整闭环。
4. 为用户提供清晰的 API 接入文档、请求示例和响应示例，降低接入成本。

## 3. 范围与边界

### 3.1 In Scope

1. 集成列表中新增“自定义API模版”卡片入口。
2. 自定义 API 模版详情页的配置、指标两个工作区。
3. 组织选择、API 秘钥展示与缺失提示。
4. 监控资产的新建、选择已有资产、导入新资产、清空与移除。
5. API 端点、请求示例、参数说明、响应示例展示。
6. 指标分组展示、折叠展开、添加指标入口与指标表单。

### 3.2 Out of Scope

1. 实际 API 接口后端实现、鉴权服务实现和数据写入逻辑。
2. API 秘钥创建流程本身，仅保留跳转或提示，不在本需求中实现。
3. 模版版本管理、发布审批、审计日志。
4. 指标公式执行引擎、数据校验规则细节、上报数据清洗逻辑。
5. 资产导入文件解析、预校验和入库处理的底层技术实现细节。

## 4. 功能需求

### 4.1 集成列表新增 API 模版入口

1. 系统应在“集成”列表中新增“自定义API模版”卡片，作为自定义接入模版的一种类型进行展示。
2. 卡片应展示模版名称、模版说明和模版标签，至少包括对象标签、接入类型标签和自定义标识。
3. 卡片应提供“接入”操作入口，用于进入自定义 API 模版配置页。
4. 自定义模版卡片应提供卡片级操作菜单，支持编辑和删除入口。
5. 自定义 API 模版应与内置模版共同出现在集成列表中，但在展示标签上应明确区分“自定义”属性。

### 4.2 模版详情页结构

1. 进入自定义 API 模版页面后，页面头部应展示模版名称和模版说明。
2. 页面左侧应提供导航菜单，至少包含“配置”“指标”两个工作区。
3. 页面左侧应提供返回入口，支持返回集成列表页面。
4. 页面默认进入“配置”工作区。
5. 用户切换左侧菜单时，右侧内容区应切换到对应工作区。

### 4.3 接入配置

1. 配置工作区应提供“组织”字段，并将其作为必填项。
2. 组织字段应支持从已有组织列表中选择。
3. 配置工作区应提供“API秘钥”展示区域，并将其作为接入前提条件进行提示。
4. 当当前组织存在 API 秘钥时，系统应自动读取并展示该组织对应的 API 秘钥。
5. 当当前组织不存在 API 秘钥时，系统应给出明确提示，并提供前往系统管理创建秘钥的入口。
6. 页面应明确告知 API 鉴权 Header 的使用方式。
7. 用户切换组织后，API 秘钥展示、监控资产范围和请求示例应同步刷新。

### 4.4 监控资产管理

1. 配置工作区应提供“监控资产”配置区域，并将其作为必填项。
2. 监控资产区域应支持“新建资产”“选择已有资产”“导入新资产”“清空选择”四类操作。
3. 已选择资产列表至少应展示资产名称、资产 ID、所属组织和移除操作。
4. 当未选择任何资产时，页面应展示空状态，并提示用户先选择组织后再选择可接入资产。
5. 页面应展示当前已选资产数量及其业务含义，例如作为同一批次监控数据上报。

### 4.5 新建资产

1. 系统应支持在当前模版页面内直接新建资产。
2. 新建资产表单至少应包含“资产名称”“所属组织”两个字段。
3. 资产名称为必填项。
4. 新建资产时，当前页面已选组织应自动加入所属组织范围，且不可取消。
5. 用户可在当前组织之外额外勾选其他组织，作为该资产的可见组织范围。
6. 新建成功后，资产应自动加入“选择已有资产”列表，并自动选中到当前模版配置中。
7. 新建成功后，资产列表、已选资产统计和请求示例应同步刷新。

### 4.6 选择已有资产

1. 系统应支持从弹窗中选择已有资产接入当前模版。
2. 弹窗中仅应展示所属组织包含当前组织的资产。
3. 弹窗应支持按资产名称搜索。
4. 弹窗应支持单个勾选和全选当前筛选结果。
5. 用户确认选择后，已选资产列表应同步更新。
6. 用户取消选择时，不应影响当前已生效的资产配置。
7. 用户确认选择后，请求示例应按当前所选资产自动生成批量上报内容。

### 4.7 导入新资产

1. 系统应支持通过文件导入方式批量新增资产并接入当前模版。
2. “导入新资产”入口仅用于创建并接入新资产，不承担已有资产的绑定能力。
3. 导入模板至少应包含“资产名称”“所属组织”两列。
4. 所属组织列用于填写资产的可见组织范围，且应包含当前页面已选组织。
5. 导入方式应支持点击上传和拖拽上传两种交互。
6. 导入文件格式应限制为 xlsx。
7. 导入文件大小应限制在 20MB 以内。
8. 当文件格式或大小不符合要求时，系统应阻止继续导入并给出明确提示。
9. 导入弹窗应提供“下载模板”入口。
10. 用户上传合法文件后，系统应先进行导入预检，并弹出结果提示，明确展示“可新增 N 个”“不支持新增 N 个”。
11. 预检结果区域应支持展开异常资产列表，至少展示异常资产在 xlsx 中的行号、资产名称、所属组织和不支持原因。
12. 不支持原因至少应覆盖：资产名称重复、所属组织未填写当前页面选择的组织、所属组织包含用户无权限的组织。
13. 用户点击确认后，应仅新增预检通过的资产；不支持新增的资产不应被创建。
14. 用户导入成功后，导入生成的资产应自动加入当前组织下的资产列表，并自动加入当前模版已选资产。

### 4.8 API 接入说明

1. 配置工作区应展示固定的 API 端点信息，并支持一键复制。
2. 页面应展示多语言请求示例，至少包括 cURL、Python、JavaScript 三种示例。
3. 请求示例应支持切换语言标签。
4. 每种请求示例应支持单独复制。
5. 请求示例中的组织信息应与当前页面所选组织保持一致。
6. 当当前组织存在 API 秘钥时，请求示例应自动带入对应秘钥。
7. 当当前组织不存在 API 秘钥时，请求示例中应保留明显的创建提示占位，而不是展示空值。
8. 当页面已选择资产时，请求示例应按已选资产生成批量上报 payload。
9. 当页面未选择资产时，请求示例应提供占位数据，提示用户先完成资产选择。
10. 页面应展示请求参数说明，至少包括模版标识、组织、时间戳、资产数组、资产指标数组等核心字段。
11. 页面应展示响应示例，至少包含成功响应和鉴权失败响应两类示例。

### 4.9 指标管理

1. 页面应提供“指标”工作区，用于维护当前自定义 API 模版的指标定义。
2. 指标工作区应支持按指标名称搜索。
3. 指标工作区应提供“添加组”入口。
4. 指标工作区应提供“添加指标”入口。
5. 页面应按指标分组展示指标列表。
6. 指标分组应支持折叠和展开。
7. 指标分组应提供编辑和删除入口。
8. 指标分组应支持拖拽排序。
9. 指标列表至少应展示以下字段：指标 ID、指标名称、维度、数据类型、单位、描述、操作。
10. 单个指标应支持编辑、删除和拖拽排序。

### 4.10 添加指标

1. 系统应支持通过弹窗添加指标。
2. 添加指标表单至少应包含“ID”“名称”“组”“维度”“公式”“数据类型”“描述”字段。
3. 其中 ID、名称、组、公式、数据类型为必填项。
4. 维度为可选项，并应支持动态增加多个维度输入项。
5. 数据类型至少应支持“数字”“枚举”两种类型。
6. 描述为可选项。
7. 用户确认添加后，新增指标应归入所选分组中展示。

### 4.11 业务规则与显示规则

1. 自定义 API 模版配置应以组织为核心上下文，页面中的秘钥、资产范围、请求示例均应受当前组织影响。
2. 当前组织下不可访问的资产，不应继续保留在当前组织的已选资产结果中。
3. 已选资产发生变化后，资产汇总文案、请求示例和成功响应示例中的统计信息应同步刷新。
4. 清空选择后，已选资产列表应恢复为空状态。
5. 从已选资产列表中移除单个资产后，页面应即时反映最新结果。
6. 请求示例中的上报方式应支持一次提交多个资产的监控数据。
7. “导入新资产”与“选择已有资产”应为两条独立操作链路，避免在单一入口中同时承担新建与已有资产绑定两类职责。
8. “导入新资产”应采用“上传后先预检、用户确认后再新增”的两阶段流程，避免合法性未知的资产被直接创建。

## 5. 验收口径

1. 在集成列表中可看到“自定义API模版”卡片，卡片展示自定义标识，并可通过“接入”进入详情页。
2. 进入详情页后，默认展示“配置”工作区，并可在“配置”“指标”之间切换。
3. 切换组织后，页面中的 API 秘钥展示、已选资产范围和请求示例应同步更新。
4. 当组织没有可用 API 秘钥时，页面应明确提示“未创建 API秘钥”或同等语义内容，并提供前往系统管理创建的入口。
5. 通过“新建资产”创建资产后，该资产应自动出现在已选资产列表中。
6. 通过“选择已有资产”弹窗选择资产时，只能看到当前组织可访问的资产，支持搜索和全选，确认后列表同步更新。
7. 通过“导入新资产”上传非 xlsx 文件或超过 20MB 的文件时，系统应阻止导入并给出提示。
8. 通过“导入新资产”上传合法文件后，系统应先展示预检结果，明确显示可新增数量与不支持新增数量，并支持展开查看异常资产的行号、资产名称、所属组织和原因；用户确认后，仅新增预检通过的资产。
9. 请求示例可在 cURL、Python、JavaScript 之间切换，并支持复制；示例中的组织、秘钥占位和资产 payload 与当前页面状态保持一致。
10. 指标工作区应展示指标分组列表，支持分组折叠展开，并提供添加指标入口。
11. 打开“添加指标”弹窗后，应能录入必填字段并完成新增；新增完成后指标应展示在对应分组下。

## 6. 约束说明

1. 本需求文档只定义用户可见能力、业务规则和验收口径，不限定前端技术实现方式。
2. 需求口径严格以当前原型表现为准；原型未体现的批量删除、版本回滚、字段级校验细则不纳入本次范围。
3. API 秘钥创建流程依赖系统管理模块，本需求仅负责读取、提示和跳转，不负责秘钥生命周期管理。
4. 若后续需要支持更多接入协议、更多编程语言示例或复杂指标计算规则，应另行补充需求文档。
