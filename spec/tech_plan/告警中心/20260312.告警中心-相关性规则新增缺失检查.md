# 告警中心-相关性规则新增缺失检查 Tech Plan（2026-03-20）

需求文档：[spec/requirements/告警中心/20260312.告警中心-相关性规则新增缺失检查.md](spec/requirements/告警中心/20260312.告警中心-相关性规则新增缺失检查.md)
方案文档：[spec/solution_plan/告警中心/20260312.告警中心-相关性规则新增缺失检查.md](spec/solution_plan/告警中心/20260312.告警中心-相关性规则新增缺失检查.md)

技术目标：在现有相关性规则体系内新增 `missing_detection` 策略类型，复用现有告警模型、现有周期任务、现有匹配器，完成“监听正常事件缺失 -> 生成告警 -> 收到心跳后自动恢复”的闭环。
非目标：不新增独立 Celery Beat、不新增 Alert 字段、不扩展到重复提醒和复杂时序编排。
影响范围：仅改动告警中心策略配置与周期处理链路，边界限定在 `server/apps/alerts` 和 `web/src/app/alarm`。

```python
from typing import Any, Literal

@dataclass(slots=True)
class MatchCondition:
    """单个条件项。外层 OR，内层 AND，直接复用现有 match_rules 结构。"""
    key: str
    operator: str
    value: Any

@dataclass(slots=True)
class MissingAlertTemplate:
    """合成告警模板。三个字段全部由用户自定义，且全部必填。"""
    title: str
    level: str
    description: str

@dataclass(slots=True)
class MissingHeartbeatContext:
    """最近一次成功心跳的最小模板变量上下文。"""
    service: str | None = None
    location: str | None = None
    resource_name: str | None = None
    resource_id: str | None = None
    resource_type: str | None = None
    item: str | None = None
    title: str | None = None
    level: str | None = None

@dataclass(slots=True)
class MissingDetectionParams:
    """存入 AlarmStrategy.params 的缺失检查配置与运行时状态。"""
    check_mode: Literal["interval", "cron"]
    interval_value: int | None = None
    interval_unit: Literal["minutes", "hours", "days"] | None = None
    cron_expr: str | None = None
    grace_period: int = 0
    activation_mode: Literal["first_heartbeat", "immediate"] = "first_heartbeat"
    auto_recovery: bool = True
    heartbeat_status: Literal["waiting", "monitoring", "alerting"] = "waiting"
    last_heartbeat_time: str | None = None
    last_heartbeat_context: MissingHeartbeatContext | None = None
    alert_template: MissingAlertTemplate | None = None
```

规则口径：
- `last_execute_time` 是策略执行时间，属于 `AlarmStrategy` 模型字段，所有策略类型共用。
- `last_heartbeat_time` 是最近一次命中 `match_rules` 的 `Event.received_at`，存入 `params`，只用于缺失检查。
- `last_heartbeat_time` 记录的是 `Event.received_at`，不是 `start_time`。
- 缺失告警不新增 Alert 字段，唯一定位条件固定为：
  - `rule_id = str(strategy.id)`
  - `group_by_field = "missing_detection"`
  - `fingerprint = md5("missing_detection:{strategy.id}")`
- `alert_template.title`、`alert_template.level`、`alert_template.description` 全部由用户配置，全部必填。

## 4. 核心逻辑伪代码

### 4.1 周期任务执行顺序

```text
event_aggregation_alert():
1. AggregationProcessor().process_aggregation()
2. TimeoutChecker.check_session_timeouts()
```

说明：
- 不新增独立 Celery Beat。
- 保持现有入口不变，不新增新的聚合调用口。
- 保持 `AggregationProcessor.process_aggregation()` 既有主流程不变，只在策略处理处增加 `strategy_type == missing_detection` 的条件判断。
- 任务顺序保持为：聚合 -> 会话超时检查。

### 4.2 单策略检查流程

```text
AggregationProcessor.process_aggregation():
1. 保持现有 process_aggregation 主流程和遍历方式不变。
2. 在原有策略遍历过程中继续调用 _process_strategy(strategy, now)。
3. 在 _process_strategy 内新增一个条件判断：
    - strategy_type == smart_denoise: 保持原聚合逻辑不变
    - strategy_type == missing_detection: 直接在当前函数内执行缺失检查逻辑，或委托给可选的私有辅助函数 `_process_missing_detection_strategy(strategy, now)`
4. 除缺失检查新增逻辑外，原有聚合、窗口、自愈、告警更新逻辑不改写。

_process_missing_detection_strategy(strategy, now):
0. 该函数不是强制要求；如果不拆函数，则以下逻辑直接内联到 `_process_strategy` 的 missing_detection 分支。
1. 使用 select_for_update() 锁定当前 strategy 行，避免并发更新 params。
2. params = _load_params(strategy)。
3. candidates = _query_candidate_events(strategy, now)。
   - strategy.last_execute_time 存在: Event.received_at > strategy.last_execute_time
   - strategy.last_execute_time 不存在: Event.received_at >= strategy.created_at
4. matched = StrategyMatcher.match_events_to_strategy(candidates, strategy.match_rules)。
5. latest_event = matched.order_by("-received_at").first()

6. 激活逻辑：
   - activation_mode == "first_heartbeat" 且 heartbeat_status == "waiting"
     - latest_event 不存在: 仅更新 last_execute_time=now，结束本轮
     - latest_event 存在:
       - heartbeat_status = "monitoring"
       - last_heartbeat_time = latest_event.received_at.isoformat()
       - last_heartbeat_context = 从 latest_event 提取模板变量上下文
   - activation_mode == "immediate" 且 heartbeat_status == "waiting"
     - heartbeat_status = "monitoring"

7. 如果 latest_event 存在：
   - 更新 last_heartbeat_time = latest_event.received_at.isoformat()
   - 更新 last_heartbeat_context
   - 如果当前 heartbeat_status == "alerting" 且 auto_recovery == True：
     - active_alert = SyntheticAlertBuilder.find_active_alert(strategy)
     - active_alert.status = AUTO_RECOVERY
     - 保存 active_alert
     - heartbeat_status = "monitoring"
   - 保存 params 和 last_execute_time，结束本轮

8. 计算 deadline：
   - anchor_time 优先取 last_heartbeat_time
   - 若 anchor_time 不存在且 activation_mode == "immediate"，取 strategy.created_at
   - 若 anchor_time 不存在且 activation_mode == "first_heartbeat"，直接结束，不做缺失判断

   interval 模式：
   - deadline = anchor_time + interval_value(interval_unit) + grace_period(minutes)

   cron 模式：
   - next_expected = croniter(cron_expr, anchor_time).get_next(datetime)
   - deadline = next_expected + grace_period(minutes)

9. 触发逻辑：
   - now <= deadline: 仅更新 last_execute_time=now，结束
   - now > deadline:
     - heartbeat_status == "alerting": 不重复创建告警，仅更新 last_execute_time
     - 否则:
       - _trigger_missing_alert(strategy, params, now)
       - heartbeat_status = "alerting"
       - 保存 params 和 last_execute_time
```

### 4.3 合成告警生成规则

```text
_trigger_missing_alert(strategy, params, now):
1. fingerprint = md5("missing_detection:{strategy.id}")
2. 按以下条件查询是否已有活跃缺失告警：
   - rule_id = str(strategy.id)
   - group_by_field = "missing_detection"
   - fingerprint = fingerprint
   - status in AlertStatus.ACTIVATE_STATUS
3. 已存在：直接返回，不新增记录
4. 不存在：
   - context = params.last_heartbeat_context or {}
   - title = render_template(params.alert_template.title, context)
   - description = render_template(params.alert_template.description, context)
   - level = params.alert_template.level
   - 创建 Alert：
     - alert_id = "ALERT-" + uuid
     - status = UNASSIGNED
     - level = level
     - title = title
     - content = description
     - first_event_time = now
     - last_event_time = now
     - fingerprint = fingerprint
     - group_by_field = "missing_detection"
     - rule_id = str(strategy.id)
     - team = strategy.dispatch_team
   - 不写入 events 关联
```

### 4.4 序列化器校验规则

```text
_validate_missing_detection(attrs):
1. match_rules 必须非空，禁止 ALL 监听
2. params.check_mode 必须是 interval 或 cron
3. interval 模式：
   - interval_value 必填且 > 0
   - interval_unit 必填且只能是 minutes/hours/days
   - cron_expr 为空或缺省
4. cron 模式：
   - cron_expr 必填
   - croniter.is_valid(cron_expr) 必须为 True
   - interval_value / interval_unit 为空或缺省
5. grace_period 必填且 > 0
6. activation_mode 必须是 first_heartbeat 或 immediate
7. alert_template.title / level / description 全部必填
8. heartbeat_status / last_heartbeat_time / last_heartbeat_context 为运行时字段：
   - create 时忽略客户端传值，后端强制初始化
   - update 时忽略客户端传值，保留数据库运行态
```

### 6.3 测试方案

后端测试文件：
- `server/apps/alerts/test/test_relevance_rule.py`

后端最小测试集：
- `AggregationProcessor.process_aggregation()` 保持原有主流程不变时，`missing_detection` 仅通过新增条件判断进入缺失检查分支，`smart_denoise` 仍走原聚合分支。
- 无论缺失检查逻辑是内联在 `_process_strategy` 中，还是提取到可选私有辅助函数，行为结果必须一致。
- `first_heartbeat` 模式下，无首条心跳时不触发告警。
- `first_heartbeat` 模式下，收到首条心跳后进入 `monitoring`。
- `immediate` 模式下，超过 `interval + grace_period` 触发 1 条告警。
- `cron` 模式下，按 `next_expected + grace_period` 触发告警。
- `alerting` 状态下再次执行不会重复创建告警。
- 收到匹配心跳且 `auto_recovery=True` 时，告警状态变为 `AUTO_RECOVERY`。
- `auto_recovery=False` 时，收到心跳不自动恢复。
- `last_heartbeat_time` 写入的是 `Event.received_at`，不是 `start_time`。
- `last_execute_time` 每轮执行后都会更新。
- serializer 对以下输入报错：
  - 空 `match_rules`
  - `grace_period <= 0`
  - interval/cron 同时填写
  - `alert_template` 三字段任一缺失
  - 非法 cron 表达式
