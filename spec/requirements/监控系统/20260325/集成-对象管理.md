# 监控系统 集成-对象管理需求

- 文档日期：2026-03-25
- 所属模块：监控系统 / 集成 / 对象
- 需求类型：新增功能
- 参考原型：spec/prototype/监控系统/集成/对象.html

## 1. 背景与问题

- 当前监控系统集成侧已有集成、资产、分组等管理入口，但缺少统一的对象管理能力，无法集中维护监控对象分类、对象定义和展示状态。
- 随着内置对象与自定义对象数量增加，需要明确对象类型、对象、子对象之间的管理关系，统一业务口径。
- 现有原型已经体现“对象类型 -> 对象 -> 子对象”的管理模型，需要沉淀成可执行、可验收的需求文档，供产品、研发、测试对齐。

## 2. 目标

1. 在监控系统“集成”模块下提供独立的“对象”管理页面，用于统一维护监控对象定义。
2. 支持对象类型和对象的新增、编辑、删除、排序、搜索等基础管理能力。
3. 支持区分内置对象与自定义对象，并对两类对象提供不同的操作边界。
4. 支持对象可见性和子对象关系配置，为后续接入、资产归类和监控展示提供对象底座。

## 3. 范围与边界

### 3.1 In Scope

1. 对象管理页面入口、导航和布局。
2. 对象类型列表展示、搜索、新建、编辑、删除、排序。
3. 当前对象类型下对象列表展示、搜索、新建、编辑、删除、排序。
4. 对象来源标识、可见性开关和空状态展示。
5. 对象图标选择和子对象配置。

### 3.2 Out of Scope

1. 采集模板、插件接入参数、资产实例详情的维护逻辑。
2. 对象字段模型设计、字段级联动规则、对象属性表单扩展。
3. 批量导入导出、细粒度权限模型、审计日志。
4. 子对象自动发现的采集实现和运行编排。

## 4. 功能需求

### 4.1 页面结构与导航

1. 系统应在监控系统“集成”模块下提供“对象”页签，与“集成”“资产”“分组”并列展示。
2. 页面应采用左侧对象类型导航、右侧对象列表工作区的双栏布局。
3. 左侧区域应展示对象类型列表，右侧区域应展示当前选中对象类型下的对象内容。
4. 页面进入后应默认选中一个对象类型，并同步展示对应对象列表。
5. 右侧标题应始终展示当前选中的对象类型名称。
6. 当当前选中对象类型为自定义类型时，右侧标题区域应展示“编辑对象类型”“删除对象类型”操作入口。
7. 当当前选中对象类型为内置类型时，右侧标题区域不应展示类型编辑和删除入口。

### 4.2 对象类型管理

1. 系统应展示对象类型的图标、名称和当前对象数量。
2. 系统应支持按关键字搜索对象类型。
3. 系统应支持新建对象类型，表单至少包含“类型ID”“类型名称”两个必填项。
4. 类型ID应作为对象类型唯一标识，不可与已有对象类型重复。
5. 用户新建的对象类型来源应标记为“自定义”。
6. 系统应支持编辑自定义对象类型的基础信息。
7. 系统不应允许编辑内置对象类型。
8. 系统应支持通过拖拽调整对象类型顺序。
9. 系统应支持删除自定义对象类型。
10. 当对象类型下仍存在对象时，系统应阻止删除，并给出明确提示。
11. 删除对象类型前，系统应进行二次确认。
12. 删除当前选中对象类型后，系统应自动切换到相邻可用对象类型；当无剩余对象类型时，右侧区域应恢复为空状态标题。

### 4.3 对象列表展示与查询

1. 对象列表应仅展示当前选中对象类型下的对象。
2. 系统应支持按关键字搜索当前对象类型下的对象。
3. 对象列表至少应展示以下字段：对象名称、对象ID、子对象数、来源、可见状态、操作。
4. 对象名称区域应同时展示对象图标与对象名称。
5. 对象ID应以清晰可识别的唯一标识形式展示。
6. 子对象数应展示当前对象已配置的子对象数量；未配置子对象时应展示为空占位。
7. 来源应明确区分“内置”和“自定义”。
8. 可见状态应以开关形式展示，并支持在列表内直接切换。
9. 系统应支持通过拖拽调整当前对象类型下对象的顺序。
10. 当当前对象类型下暂无对象时，页面应展示空状态，并提供新建对象的引导。

### 4.4 对象新建与编辑

1. 系统应支持在当前选中的对象类型下新建对象。
2. 新建和编辑对象的表单至少应包含“对象ID”“对象名称”“图标”“子对象”四类信息。
3. 对象ID和对象名称为必填项。
4. 对象ID应作为对象唯一标识，不可与已有对象重复。
5. 用户新建的对象来源应标记为“自定义”。
6. 图标应支持从预设图标集中进行选择。
7. 新建对象时，对象归属的对象类型应默认为当前选中对象类型。
8. 编辑对象时，系统应回显该对象已有配置。
9. 系统应允许编辑自定义对象。
10. 系统不应允许编辑内置对象定义。

### 4.5 子对象配置

1. 对象表单中的子对象配置应为可选项。
2. 系统应支持为一个对象配置多个子对象。
3. 每个子对象至少应包含“子对象名称”“子对象ID”两项信息。
4. 系统应支持在对象表单内动态新增子对象行。
5. 系统应支持删除未保存或已配置的子对象行。
6. 对象列表中的“子对象数”应与对象配置的实际数量保持一致。
7. 子对象用于描述监控主对象过程中自动发现的关联对象。

### 4.6 来源与操作限制

1. 内置对象类型仅支持展示、搜索、排序，不支持编辑和删除。
2. 自定义对象类型支持编辑、删除、排序。
3. 内置对象支持可见性切换，但不支持编辑和删除。
4. 自定义对象支持编辑、删除、可见性切换和排序。
5. 系统不应允许用户在列表页直接修改对象来源。

### 4.7 状态与显示规则

1. 对象类型上的数量应反映该类型下当前对象总数。
2. 切换对象类型后，右侧标题、对象列表和操作区应同步刷新。
3. 切换对象可见性后，页面应即时反映最新状态。
4. 删除对象后，对象列表和所属对象类型计数应同步更新。
5. 删除对象类型后，左侧对象类型列表应同步更新。

## 5. 验收口径

1. 进入“集成-对象”页面时，左侧应显示对象类型列表，右侧应显示当前选中对象类型的名称和对象列表；切换对象类型后，右侧内容应同步变化。
2. 新建对象类型时，输入合法且不重复的类型ID、类型名称后可创建成功；任一必填项缺失或类型ID重复时不可提交。
3. 选中内置对象类型时，不显示类型编辑和删除入口；选中自定义对象类型时，显示对应入口。
4. 删除包含对象的自定义对象类型时，系统应阻止删除，并给出“该对象类型下存在对象，无法删除”或同等语义提示。
5. 在当前对象类型下新建对象时，应可填写对象ID、对象名称、选择图标，并支持添加多个子对象；创建成功后列表应新增对象记录，子对象数正确展示。
6. 内置对象仅可切换可见性；自定义对象可编辑、删除并切换可见性。
7. 拖拽对象类型或对象后，页面展示顺序应立即更新。
8. 当前对象类型下没有对象时，页面应展示空状态，并给出继续新建对象的引导。
9. 搜索对象类型或对象时，应仅在对应列表范围内按关键字过滤结果。

## 6. 约束说明

1. 本需求文档仅定义对象管理业务规则和页面行为，不约束具体前端实现方式。
2. 需求口径以当前原型表现为准，未在原型中体现的字段扩展和批量能力不纳入本次范围。
3. 若后续需要支持对象属性字段、采集模板绑定或对象权限拆分，应另行补充需求文档。
